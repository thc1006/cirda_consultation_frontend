# 專案關鍵抉擇紀錄

> 建立日期：2026-04-08
> 用途：保存使用者於專案啟動階段對核心技術方向的決定，作為後續 ADR、todolist、實作的上游約束。

## 1. 技術棧
- 由開發方依據 `CLAUDE.md`、`國網中心創進學院申請書.md`、`國網創進計畫_期中報告.txt`、`CBT.html` 的深度閱讀結果自行決定最適合的前端技術棧。
- 邊界條件：需支援 RWD、WCAG 無障礙、Playwright E2E、主題切換（含色盲友善配色）、可部署於本機 k8s。

## 2. LLM 後端串接
- Demo 階段：沿用 `CBT.html` 的 Puter.js 方式呼叫 `puter.ai.chat`，模型固定為 `google/gemini-3-flash-preview`。
- 未來：替換為微調後的「諮心好友 LLM」（保留可抽換的 provider 介面）。

## 3. 資料儲存
- 由開發方決定，以「方便 demo、無資安疑慮」為原則。
- 硬性要求：
  - 不得將任何金鑰、Token 寫死於前端或 commit。
  - 受試者對話、前後測量表屬個資，儲存時需避免明文外洩，匯出時需可產生對 LLM 與人類皆友善的結構化格式。

## 4. 記憶機制
- 使用者本人不熟悉，授權開發方進行深度網路調研（至少五份權威來源交叉驗證）後再設計。
- 需同時考量短期（會話內上下文）與長期（跨會話回顧）兩層記憶的最佳實踐。

## 5. 深度調研授權
- 已授權開發方放手執行大規模網路調研，現在是 2026-04-08，務必查最新資訊。
- 所有採用觀點須標註來源，至少五份權威來源共同驗證。

## 6. 流程與風格紅線
- 所有階段不可 `git push`。
- 報告檔案禁用 emoji；程式碼、註解、文件不得出現 Claude / Anthropic 相關字樣。
- 註解一律繁體中文口語；註解長度不得超過被註解的程式碼本身。
- TDD 全實作，不得 SKIP 任何任務；每子階段完成需做 docs/plan review 後再勾選。
